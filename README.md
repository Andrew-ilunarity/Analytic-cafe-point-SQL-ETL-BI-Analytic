

#### Кейс: Наскрізна аналітика та побудова ETL-пайплайну для роздрібної точки продажу (Кав'ярня)

#### 📌 **Situation (Ситуація)**

Компанія (кав'ярня) накопичувала транзакційні дані про продажі в CRM-системі. Проте через технічні збої та людський фактор сирий масив даних (`dirty_cafe_sales.csv`) обсягом понад 8 500 рядків містив критичні дефекти: пропущені назви товарів, відсутні ціни за одиницю, зсуви форматів даних (наприклад, кириличні баги в датах типу `04.Тра` замість фінансових значень) та порожні поля в локаціях і методах оплати. У такому вигляді дані були повністю непридатні для аналізу, а керівництво не мало можливості відстежувати реальну виручку, середній чек (AOV) та помісячні тренди для фінансового планування.

#### 🎯 **Task (Завдання)**

Переді мною стояло завдання виступити в ролі Data Engineer та BI-аналітика, щоб самостійно реалізувати повний цикл обробки даних (End-to-End pipeline) менш ніж за 5 годин:

1. Розгорнути локальну базу даних та імпортувати сирий масив.
2. Провести глибокий аудит (Data Quality Check) та очистити дані без втрати корисної вибірки.
3. Розробити інтерактивний дашборд для менеджменту, який підсвітить ключові метрики ефективності (KPIs), структуру продажів та операційні інсайти.

#### 🛠️ **Action (Дії)**

Для досягнення мети я розбив роботу на три технічні етапи:

1. **Імпорт та інженерія даних (SQL / DBeaver):**
* Розгорнув реляційну базу даних у СУБД та успішно імпортував «брудний» CSV-файл, розібравшись зі специфічними розділювачами.
* Замість механічного видалення рядків із пропусками (що викривило б фінансову звітність), я застосував методику відновлення даних (*Data Imputation*). Наприклад, виявив закономірність вартості позиції `Tea` ($1.5) і написав `UPDATE`-запит для відновлення текстових категорій у порожніх рядках за ціновим паттерном.
* Написав математичні скрипти для регенерації ціни за одиницю через ділення загальних витрат на кількість (`co."Total Spent" / co."Quantity"`) там, де поле `Price per Unit` було порожнім.


2. **Моделювання та розрахунок метрик (Power BI / DAX):**
* Завантажив очищений на рівні сервера масив даних у Power BI.
* Написав пул аналітичних мір мовою **DAX** для розрахунку ключових бізнес-показників: загального виторгу (`SUM`), обсягу продажів у штуках та середнього чека транзакції ($AOV$).


3. **Візуалізація та Data Storytelling:**
* Створив комплексний багатовимірний дашборд. Вивів головні KPIs на інтерактивні картки для миттєвого зчитування стану бізнесу.
* Налаштував систему динамічних зрізів за категоріями, платіжними методами та локаціями.
* Побудував лінійний графік часових рядів для аналізу помісячної динаміки та кільцеві діаграми для оцінки часток ринку.



#### 🏆 **Result (Результат)**

* **Швидкість та автономність:** Повний цикл проєкту від сирого файлу до презентації готового дашборду виконано за **4 години 41 хвилину** без залучення сторонніх фахівців.
* **Якість даних:** Відновлено цілісність бази даних (100% рядків валідовані, математичні помилки розрахунків та текстові баги повністю ліквідовані).
* **Бізнес-інсайти для керівництва (знайдені за допомогою дашборду):**
* Розраховано точні показники бізнесу: загальний виторг склав **$29,070**, продано **8,561** одиницю товару, а середній чек ($AOV$) зафіксовано на рівні **$10.22**.
* Виявлено неочевидного лідера за виручкою — категорію `Sandwich` ($6,376), що підсвітило високу ефективність ситної кухні порівняно з десертами (`Cookie` принесли найменше — $1,114).
* Доведено стійкість бізнесу до сезонних коливань: щомісячний виторг стабільно тримається в межах **$2.2K – $2.6K**, з піком у березні ($2,572).
* Локалізовано операційну проблему CRM-системи: виявлено, що **$8.2K** прибутку зависли в статусі `Unknown` за локацією, що дозволило сформувати технічне завдання для розробників на впровадження обов'язкових полів при закритті чеків.
